Por que MongoDB para chat?

Mensagens são naturalmente documentos: cada uma carrega autor, conteúdo e metadados num único objeto JSON — sem JOINs.

Schema flexível permite adicionar campos (reactions, attachments, threads) sem migração de tabela.

Índice em { room, timestamp } torna consultas de scroll histórico eficientes.

Soft delete (deleted: true) preserva o histórico na coleção, padrão comum em chats reais.
